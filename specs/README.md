# specs/ — Specs de features

Output do sub-agente `product-spec`, input pros agentes de
implementação (`backend-engineer`, `frontend-engineer`,
`ai-integration`, `supabase-architect`).

Cada spec descreve a feature e os critérios de aceitação e, quando a
feature toca dado sensível, os requisitos de LGPD/RLS a validar
depois com o `security-reviewer`.

Convenção de nome sugerida: `YYYY-MM-DD-nome-da-feature.md`.
